KLO App Store Guide 3 / 9
Step 1

Deploy to AWS Amplify

This is the server that powers the app. The native apps load from this URL.

  1. Log in to AWS Console → search for Amplify
  2. Click "New app" → "Host web app"
  3. Connect your GitHub account and select the klo repository
  4. Select the main branch
  5. Amplify auto-detects Next.js — accept the default build settings
  6. Click "Save and deploy" — wait for the build to complete (~3-5 min)
  7. Go to "Domain management" → add keithlodom.ai
  8. Update your DNS to point to the Amplify URL (Amplify shows you exactly what records to add)

Environment Variables

In Amplify → App settings → Environment variables, add these:

NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY
SUPABASE_SERVICE_ROLE_KEY
NEXTAUTH_SECRET
NEXTAUTH_URL
GOOGLE_CLIENT_ID
GOOGLE_CLIENT_SECRET
STRIPE_SECRET_KEY
STRIPE_PUBLISHABLE_KEY

Tim will send you the values for each of these securely.

KLO App Store Guide 4 / 9
Step 2A

Google Play Store Submission

Publish the Android version of the KLO app.

  1. Go to play.google.com/console — sign in with your developer account
  2. Click "Create app" → Name: Keith L. Odom, English, App, Free
  3. "Main store listing" → short desc, full desc, screenshots, feature graphic, icon (Tim provides all assets)
  4. Complete "Content rating" — informational, no violence/gambling
  5. "Target audience" → 18+ (not directed at children)
  6. "App content" → privacy policy: keithlodom.ai/privacy
  7. Production → "Create new release" → upload .aab file from Tim
  8. Release notes: "Initial release of the Keith L. Odom app"
  9. "Review release" → "Start rollout to Production"

KLO App Store Guide 5 / 9
Step 2B

Apple App Store Submission

Publish the iOS version of the KLO app.

  1. Go to appstoreconnect.apple.com — sign in with your Apple Developer account
  2. Click "+" → "New App" — iOS, Name: Keith L. Odom, Bundle ID: io.keithlodom.klo, SKU: klo-app-001
  3. Register Bundle ID first in Certificates, Identifiers & Profiles
  4. "App Information" — Privacy: keithlodom.ai/privacy, Support: keithlodom.ai/support, Category: Business
  5. "1.0 Prepare for Submission" — upload screenshots (Tim provides at 6.7", 6.5", iPad sizes)
  6. Add description, keywords, marketing URL
  7. Under "Build" — select the build Tim uploaded via Xcode
  8. "App Review Information" — note that guests join events without sign-in
  9. Click "Submit for Review"

Review Time

First submission: 1-3 days

90% within 24 hours

Important

Tim needs temporary access to your Apple Developer account to upload the build, OR you can run the Xcode archive on a Mac.

Review Notes

Tell Apple that guests join events without sign-in — no demo account needed.
